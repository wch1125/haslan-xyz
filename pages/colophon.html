Colophon

Abstract: How this site is built, what inspired it, and why I made certain choices. Part craft explanation, part invitation to steal liberally.

This is a static site with no build step, no CMS, and no framework. Just HTML, CSS, and vanilla JavaScript. Everything is version-controlled in Git and hosted on a simple server.

I wanted something I could understand completely, fix when it breaks, and keep running for decades if needed. The web has a long memory; platforms don't.

Design Inspirations

The visual and interaction design draws from several sources:

  • Gwern.net — The gold standard for personal wikis. Inspired the overall structure, the use of sidenotes, and the "evergreen" content philosophy.
  • Tufte CSS — The marginal notes pattern and the idea that typography can do heavy lifting.
  • C.S. Lewis's Narnia — The name "H. Aslan" and the subtle lion motif throughout (gold accents, the loading animation, console easter eggs). Those who know will recognize it; everyone else just sees a clean design.

Key Features

Defined Terms System

Capitalized terms like Personal Domain automatically link to the glossary and show hover previews. This creates a web of interconnected concepts—more like a concept map than a traditional site.

Sidenotes

Marginal notes appear on desktop.Like this one. They let you add context without interrupting the main flow. On mobile, they collapse inline. Inspired by Tufte's principle that footnotes interrupt reading, but side-by-side content doesn't.

Progressive Disclosure

Collapsible sections let readers control depth. Show the overview first; details on demand.

Like this

If you're curious, you click. If not, you keep reading. Either choice is valid.

Dark Mode

Respects system preference, remembers your choice, transitions smoothly. Press Alt+D to toggle.

Tools & Stack

  • HTML/CSS/JS — No frameworks. I want to understand every line.
  • Git — Version control for everything. Every change is tracked.
  • Python (Flask) — A small wiki-manager script for adding content locally.
  • Any text editor — I mostly use VS Code, but the site doesn't care.

Total dependencies: effectively zero. The site will work as long as browsers render HTML.

Typography

  • Body text: System serif stack (Iowan Old Style → Palatino → Georgia)
  • UI elements: System sans-serif stack
  • Code: SF Mono → Consolas → Monaco

No web fonts to load. Fast, reliable, and the reader's system handles rendering.

Color Palette

Mostly monochrome with one accent color: lion's mane gold (#B8860B). The gold appears in dropcaps, hover states, the loading animation, and the progress bar. It's the only "brand" color.

The restraint is intentional. Every bit of gold should feel earned.

Steal This

If you want to make something similar, go ahead. The patterns here aren't original—I borrowed heavily from the sources above. Personal websites are collaborative even when we don't talk to each other; we learn from what we see and iterate.

If you want help setting up your own, reach out. I'm happy to point you at resources or answer questions.

Starting from scratch?

A few places to begin:

  • Neocities — Free hosting, no coding required to start
  • GitHub Pages — Free hosting if you're comfortable with Git
  • Bear Blog — Simple blogging, no distractions

Start small. A single HTML page with your name and a paragraph is a personal website. Everything else is iteration.
